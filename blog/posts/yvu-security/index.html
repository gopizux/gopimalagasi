I Found a Massive Security Gap in Yogi Vemana University (www.yvu.edu.in) — And No Login Was Required to Grab Student Data

July 18, 2026

During a recent penetration test on a state university's web infrastructure, I stumbled across some alarming security gaps. I want to share what I found to highlight how simple oversights can put real people at risk—but I'm intentionally leaving out specific names, exact URLs, and sensitive data to protect the students involved.

How It Started

I was doing some routine work on a state university here in India. Like a lot of educational institutions, they've built out a pretty large web presence with different subdomains handling their main site, online education portals, and examination systems.

I started with the basics: just mapping out their attack surface to see what was publicly visible. Within minutes, I had a good picture of their setup. They were running a pretty standard tech stack—Apache, PHP, and the CodeIgniter framework. Nothing unusual at all.

Then, I started digging a little deeper, and things got messy.

What I Found

1. A Student Database Left Wide Open

This was the one that made my jaw drop.

While exploring their online education portal, I noticed an endpoint designed to let super admins export student data into a spreadsheet. The massive flaw? There was absolutely no login required to access it.

I ran a single command to request that page, and it immediately downloaded a file to my computer. It contained records for over a hundred students.

I won't go into the exact fields, but the file contained deeply personal information. I'm talking full names, phone numbers, home addresses, family details, and complete education histories. Worst of all, it included highly sensitive national ID numbers (similar to a Social Security Number).

Let that sink in for a second. With one unauthenticated click, anyone on the internet could have downloaded enough information to ruin over a hundred people's lives through identity theft.

When I looked at how this could happen, the answer was depressingly simple. In the source code, the developer had written a function to pull all the student records from the database and serve them up, but they simply forgot to add the line of code that checks if the user is actually logged in. That's it. One missing line of code.
